# 04-06 Island Reversal（島型反轉）

## 策略類型
**04 — Pattern Trading（型態學 / 極端反轉）**

---

## 概述

島型反轉（Island Reversal）是技術分析中**最為罕見且強力的反轉訊號之一**。其形態由**兩個相反方向的跳空缺口（Gap）**圍住一段整理區域，使這段整理區在圖表上看起來像一個孤立的「島嶼」。

這種型態代表：原本強勢順勢的籌碼被**瞬間全數套牢**（頂部）或**全數軋空**（底部），市場在極短時間內完成了多空力量的徹底易主。

---

## 型態結構

### 頂部島型反轉（做空型態）
```
         島嶼（整理區）
       ╔══════════════╗
       ║              ║
——→ 跳空高開         跳空低開 ——→ 繼續下跌
（缺口1：向上）       （缺口2：向下）
```

**形成過程：**
1. 強勢上漲後 → **跳空高開（Gap Up）** 進入整理
2. 整理區間維持 1 至數日（形成「島嶼」）
3. **跳空低開（Gap Down）** 跌破整理區 → 確認反轉

### 底部島型反轉（做多型態）
```
（缺口1：向下）跳空低開 → 整理 → 跳空高開（缺口2：向上）→ 持續上漲
              ╚══════════════╝
                   島嶼
```

---

## 關鍵特徵 (量化嚴謹版)

| 特徵 | 說明 | 量化定義 (標籤: `Param_Island`) |
|------|------|------|
| **雙缺口大小** | 極端跳空 | `Gap1 > 1.0 * ATR` 且 `Gap2 > 1.0 * ATR` |
| **島嶼時間 (T)** | 孤島持續期 | `1 <= T_island <= 10 bars` (過長則失去爆發力) |
| **成交量驗證** | 縮量島嶼 | 島嶼內平均成交量應小於趨勢期均量的 70% |
| **反轉確認** | 跳空離開 | `Open[今日]` 必須跳離島嶼最低(高)點 |

---

## 交易規則 (標籤: `Island_Execution`)

### 進場策略 (以頂部島型為例)
- **觸發點**：當出現第二個向下缺口，且價格低於島嶼區間最低價時進場。
- **過濾條件**：若 `RSI(14) > 80` (超買) 時出現，勝率顯著提高。

### 停損與目標
- **停損 (SL)**：島嶼的最高點 + `0.5 * ATR`。
- **目標 (TP)**：回測趨勢起點或利用 `ATR` 定義 3 倍風報比。

---

## 量化優化方向 (Optimization Hooks)

1. **參數優化**：
   - **Gap_Multiplier**: [0.5, 1.0, 1.5] (決定缺口的極端程度)。
   - **Max_Island_Days**: [3, 5, 10, 20] (島嶼維持多久算失效)。
2. **PCA 與特徵分析**：
   - 分析 **「島嶼內部的 K 線重疊度」** 與後續反轉力道的相關性。重疊度越高（盤整越久）通常反轉越穩。
3. **實作細節**：
   - 在加密貨幣等 24/7 市場，原生跳空缺口較少，建議使用 **「收盤價與次根 K 線開盤價的偏移量」** 或 **「1 分鐘內的瞬時爆量位移」** 來模擬島型結構。

---

## 量化研究補充（必要條件 / 可選條件）

- 必要條件：需先確認市場存在可辨識跳空，對 24/7 加密市場應改成流動性真空或急速脫離區間的代理定義。
- 必要條件：兩側缺口都必須以已完成 K 線確認，且明確定義是否允許部分回補。
- 必要條件：回測需納入開盤跳空後的滑點，否則島型反轉的理論風報比會被高估。
- 可選條件：將缺口大小、孤島停留時間、孤島區間量能、第二缺口方向加速度做成 feature。
- 可選條件：加入事件因子，區分財報型 gap、消息型 gap、宏觀 shock gap。
- 可選條件：將島型反轉與竭盡缺口回補合併為同一組 extreme reversal family 共同優化。

## 為何有效？

**套牢籌碼的心理動力：**
- 在跳空缺口形成時進入島嶼的買方（或賣方）
- 被反向缺口套牢後，面臨平倉壓力
- 這些平倉行為進一步推動反轉方向的動能

**加密市場的特殊情況：**
- 幣安（Binance）等交易所在重大消息後（如 SEC 訴訟、監管公告）常出現島型反轉
- 週末低流動性期間的消息面衝擊容易形成缺口

---

## 關鍵指標

| 指標 | 用途 |
|------|------|
| **跳空缺口大小** | 相對於 ATR 的倍數，越大越極端 |
| **成交量** | 確認缺口的「真實性」 |
| **RSI** | 頂部島型時 RSI 通常 > 75；底部島型 < 25 |
| **前趨勢長度** | 趨勢越長，島型反轉的可信度越高 |

## 交易規則

### 頂部島型做空
```
1. 確認形成：出現向上跳空（缺口1），隨後整理 1-N 天
2. 觸發：出現向下跳空（缺口2），跳空低於島嶼最低價
3. 進場：缺口2 形成的當日收盤前進場做空
   （或次日確認缺口未回填後做空）
4. 停損：島嶼最高點（缺口1 的上方）
5. 目標：趨勢起始點（形成前的上漲起點）
```

### 回測填補缺口策略
```
部分交易者等待「回測缺口（Gap Test）」：
- 反向缺口形成後，價格短暫嘗試填補缺口
- 若填補失敗（無法站回島嶼）→ 更強的確認進場點
- 這種進場勝率更高但可能錯過部分波段
```

## 量化實作

```python
def detect_island_reversal(df, trend_min_bars=20, trend_min_return=0.10,
                            gap_min_size=0.01):
    """
    檢測頂部島型反轉
    """
    signals = []

    for i in range(trend_min_bars + 5, len(df) - 1):
        # 確認之前有上升趨勢
        trend_start_close = df['close'].iloc[i - trend_min_bars]
        island_start_open = df['open'].iloc[i - 4]  # 島嶼起始
        trend_return = (island_start_open - trend_start_close) / trend_start_close

        if trend_return < trend_min_return:
            continue

        # 缺口1：向上跳空進入島嶼
        gap1_up = df['open'].iloc[i-3] > df['high'].iloc[i-4]  # 開盤高於前高
        if not gap1_up:
            continue

        # 島嶼的最高和最低價
        island_high = df['high'].iloc[i-3:i].max()
        island_low  = df['low'].iloc[i-3:i].min()

        # 缺口2：向下跳空離開島嶼
        gap2_down = df['open'].iloc[i] < df['low'].iloc[i-1]
        if not gap2_down:
            continue

        # 確認缺口大小足夠
        gap2_size = (df['low'].iloc[i-1] - df['open'].iloc[i]) / df['close'].iloc[i-1]
        if gap2_size < gap_min_size:
            continue

        signals.append({
            'bar': i,
            'island_high': island_high,
            'island_low': island_low,
            'gap2_open': df['open'].iloc[i],
            'stop_loss': island_high,
            'entry': df['close'].iloc[i]
        })

    return signals
```

## 優缺點

| 優點 | 缺點 |
|------|------|
| 反轉訊號極強，一旦確認成功率高 | 出現頻率非常低 |
| 停損點明確（島嶼高/低點） | 需要跳空缺口，在高流動性市場較少見 |
| 套牢籌碼的心理壓力持續推動趨勢 | 部分缺口在隔日開盤被填補（週末效應） |
