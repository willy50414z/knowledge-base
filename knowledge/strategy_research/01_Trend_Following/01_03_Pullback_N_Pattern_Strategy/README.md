# 01-03 Pullback / N-Pattern Strategy（拉回做多策略）

## 策略類型
**01 — Trend Following（趨勢跟蹤）**

---

## 概述

拉回做多策略（又稱 N 型態策略）是趨勢交易中最基本且廣泛應用的進場方法。其核心理念是：**不追高，等市場在上升趨勢中自然回撤後，再順勢低接。** N 型態指的是價格走勢在圖表上形成的「~~高、低、更高、更低~~ 高點突破後拉回、再轉強上攻」波段結構，外形近似字母 N。

此策略結合了道氏理論（Dow Theory）對多頭趨勢的定義：**底底高（Higher Low）+ 頭頭高（Higher High）**。

---

## 核心邏輯

> **「趨勢已確認 → 等回調 → 確認不破前低 → 再度放量轉強時進場」**

多頭趨勢的基本結構：
```
  HL2       HH2
   /\       /
  /  \     /
HH1   \   /
 /     HL1
/
```
- **HH（Higher High）**：每波高點比前一波高
- **HL（Higher Low）**：每波低點也比前一波高（趨勢未被破壞）

---

## 關鍵指標與量化參數

| 指標 | 用途 | 量化建議 (標籤: `Param_N`) |
|------|------|------|
| **ZigZag / Local Extrema** | 自動標記波段高低點 | `Order` 參數 (預設 5-10)，決定結構敏感度 |
| **Fibonacci Retracement** | 衡量回撤深度 | 區間 `[0.382, 0.618]`，超過 0.786 視為結構失效 |
| **成交量比率** | 確認回撤量縮 | `Volume < MA(Volume, 20) * 0.8` |
| **轉強確認** | 進場觸發點 | `Close > High[前一根 K]` 或 `Close > 均線` |

---

## 交易規則 (量化嚴謹版)

### 進場條件 (全部滿足)
1. **趨勢確認 (Trend ID)**：
   - `HH[n] > HH[n-1]` 且 `HL[n] > HL[n-1]`。
   - 這裡的 `n` 代表透過 `Order` 確認後的局部極值序號。
2. **合法回撤 (Valid Pullback)**：
   - 當前價格 `P` 滿足：`HL[n] < P < HH[n]`。
   - 最佳區間：`P` 處於 `HH[n]` 到 `HL[n]` 的 Fibonacci 50%-61.8% 位置。
3. **動能轉強 (Trigger)**：
   - 價格收盤 ~~突破回撤小波段的高點~~ `Close > Max(Pullback_Highs)`。
   - 同時 `Volume > MA(Volume, 20) * 1.2` (放量確認)。

### 出場條件
| 情況 | 處理 | 優化點 |
|------|------|------|
| **停損 (SL)** | ~~收盤跌破最近 HL 點~~ `Close < HL[n]` | 可加掛 `ATR * 0.5` 的緩衝墊 |
| **獲利 (TP)** | 前高 `HH[n]` 或 `1.618 * (HH[n] - HL[n])` | 建議使用分批止盈 |

---

## 量化實作要點 (標籤: `Optimization`)

1. **極值滯後性 (Lag)**：
   `argrelextrema` 需要 `Order` 根 K 線確認，因此量化訊號會有 `Order` 期的滯後，需在回測中精確計算。
2. **PCA 優化空間**：
   - `Order` 大小 (結構粗細)
   - `Retracement_Level` (回撤深度容忍度)
   - `Volume_Threshold` (轉強放量倍數) 針對不同標的進行 PCA 分析其貢獻度。

---

## 實作挑戰

1. **如何定義「局部高低點」**：order 參數（需要多少根 K 線確認轉折）決定訊號的靈敏度
2. **回撤深度容忍**：拉回多少才算正常回撤？通常設 0.382~0.618 Fibonacci 回撤範圍
3. **假突破過濾**：需搭配量能、均線位置排除弱訊號

---

## 量化研究補充（必要條件 / 可選條件）

- 必要條件：局部極值需在右側至少確認 `Order` 根 K 線後才成立，否則 HH/HL 判定會偷看未來。
- 必要條件：建議加入趨勢過濾，例如 `EMA50 > EMA200`、`ADX > 20` 或結構高點持續上移，避免在盤整區硬做 N 型。
- 必要條件：回測需納入成交量與流動性門檻，否則「放量轉強」在小型幣或冷門股容易失真。
- 可選條件：把回撤深度、回撤天數、反彈第一根實體大小、量價比率記錄成 feature，供 PCA 與 SHAP 排序。
- 可選條件：加入事件濾網，避開財報、CPI、FOMC 或幣圈重大 unlock 前後的異常波動。
- 可選條件：測試時間停損，例如回撤後 `N` 根內未創新高則退出。

---

## 優缺點

| 優點 | 缺點 |
|------|------|
| 進場位置明確，風報比佳（不追高） | 判斷趨勢的局部極值定義影響結果極大 |
| 邏輯符合大多數技術分析框架 | 趨勢末段拉回可能演變為反轉，難以即時區分 |
| 適用所有時間框架與市場 | 震盪市場中頻繁出現假多頭結構 |
