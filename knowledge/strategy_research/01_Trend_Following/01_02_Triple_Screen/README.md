# 01-02 Triple Screen Strategy（三重濾網策略）

## 策略類型
**01 — Trend Following（趨勢跟蹤）**

---

## 概述

三重濾網策略由醫師兼交易員 **Dr. Alexander Elder** 於 1986 年在《Futures》雜誌首次發表，後收錄於其著作《Trading for a Living》。策略的核心理念是：**任何單一指標都有盲點，必須使用不同時間框架的多重過濾條件，才能有效識別高勝率進場點。**

Elder 以「海洋的潮汐、海浪與漣漪」比喻三個時間框架：
- **第一重（潮汐）**：長週期，定義大方向
- **第二重（海浪）**：中週期，尋找逆向修正
- **第三重（漣漪）**：短週期，精確進場觸發

---

## 核心邏輯

> **「順大勢（長週期），逆小勢（中週期修正）進場，用最短線（短週期）設定觸發條件。」**

只在長期趨勢方向一致的前提下，趁中期出現修正（超賣/超買）時進場，避免追高殺低。

---

## 關鍵指標

| 層次 | 建議週期 | 指標 | 用途 |
|------|---------|------|------|
| **第一重濾網** | 週線 / 日線 | MACD (12,26,9) 柱狀圖方向 | 確認大趨勢方向 |
| **第二重濾網** | 日線 / 4H線 | RSI(14) 或 KD(9,3,3) | 尋找短期超買/超賣 |
| **第三重濾網** | 1H線 / 15M線 | 突破前一根 K 線高/低點 | 精確進場觸發 |

### 時間框架乘數規則
Elder 建議相鄰層次的時間框架比例約為 **5 倍**：
- 長：中：短 ≈ **週線 : 日線 : 小時線**
- 或 **日線 : 4H線 : 1H線**

---

## 交易規則

### 做多流程 (量化邏輯)
1. **第一重 (趨勢過濾)**：
   - 週期：週線 (W1)
   - 條件：`MACD_Hist[0] > MACD_Hist[-1]` (斜率向上) 或 `MACD_Hist[0] > 0`。
   - **量化細節**：需注意週線數據在日線回測中的「未來函數」問題，務必使用已收盤的週線。

2. **第二重 (波段過濾)**：
   - 週期：日線 (D1)
   - 條件：`RSI(14) < 30` 或 `Force_Index(13)` 降至零軸以下。
   - **量化細節**：定義一個「進入超賣區」的狀態位，啟動第三重濾網監控。

3. **第三重 (進場觸發)**：
   - 週期：日線 (D1) 或更小
   - 條件：當前一根日線滿足 RSI 超賣時，設定一個 **Buy Stop 單** 於 `High[昨日] + 1 tick`。
   - **失效機制**：若當日價格未觸及該 Buy Stop，則撤單，並根據新的 `High` 重新掛單，直到 RSI 脫離超賣區。

## 停損與目標 (量化定義)

| 項目 | 規則 | 優化建議 |
|------|------|------|
| **停損 (SL)** | `min(Low[最近 N 日])` 或 `Entry - 2*ATR` | ATR 倍數可作為優化變數 |
| **目標 (TP)** | 風報比 (RR) >= 2.0 | 建議改用**移動止盈 (Trailing Stop)** |
| **退出 (Exit)** | 第一重濾網 MACD 斜率反轉 | `MACD_Hist[0] < MACD_Hist[-1]` |

---

## 量化實作要點 (標籤: `Multi_TF_Sync`)

1. **數據同步 (Alignment)**：
   在量化引擎中，需確保 `W1` 指標在 `D1` 每個 K 線開始時已經準備好。
   ```python
   # 避免 Look-ahead Bias
   df['weekly_trend'] = df['weekly_macd'].shift(1) # 使用上一週的收盤數據
   ```

2. **優化參數 (Optimization Hooks)**：
   - **指標選擇**：第二重濾網可用 RSI, KD, 或是 Williams %R 進行 PCA 貢獻度分析。
   - **超賣閾值**：RSI [20, 30, 40] 的敏感度測試。
   - **時間框架比例**：測試 4x, 5x, 6x 比例的效果是否優於標準 5 倍。

---

## 量化實作要點

```python
# 偽代碼示意（三個時間框架同時訂閱）
# 第一重：週線 MACD
weekly_macd_hist = macd(weekly_close, 12, 26, 9).histogram
# 第二重：日線 RSI
daily_rsi = rsi(daily_close, 14)
# 第三重：4H 突破確認
hourly_high = rolling_max(4h_high, 1)  # 前一根4H高點

if weekly_macd_hist > 0:            # 大趨勢多頭
    if daily_rsi < 40:              # 日線短期超賣
        if close > hourly_high[-1]: # 4H突破進場
            buy()
```

**實作挑戰：**
- 需同時訂閱並同步不同時間框架的 K 線資料流
- 時間框架對齊：確保不同週期 K 線的時間戳一致
- 換線時的資料重整（尤其是週線在週五收盤）

---

## 量化研究補充（必要條件 / 可選條件）

- 必要條件：多時間框架資料必須以各自收盤完成後才更新高階訊號，避免週線或日線未收盤時產生 future leak。
- 必要條件：需明確定義執行層的下單價格，建議使用 `next_bar_open` 或 stop order 模擬，而非直接用訊號 bar 收盤成交。
- 必要條件：回測需計入交易成本，因為多重濾網會壓低交易頻率，單筆成本占比會被放大。
- 可選條件：加入 `EMA slope` 或 `ADX` 作為第一層趨勢強度分數，而不只看 MACD 正負。
- 可選條件：把第二層超賣條件改為連續特徵，例如 `RSI percentile`、`Williams %R`、`Pullback Depth` 供 PCA 使用。
- 可選條件：加入跨商品或跨產業相對強弱，只在上位強勢群組內找回撤。

---

## 優缺點

| 優點 | 缺點 |
|------|------|
| 有效過濾逆勢交易，降低假訊號 | 多重條件同時滿足的機會較少，交易頻率低 |
| 邏輯清晰，易於理解與回測 | 長週期確認滯後，可能錯過初始大波段 |
| 結合趨勢與動能，適應性強 | 程式需處理多時間框架資料同步，開發難度中等 |
